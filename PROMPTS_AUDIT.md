# Аудит промптов «Зазеркалья»

Этот документ описывает актуальную архитектуру промптов расширения. Пользовательский интерфейс русскоязычный; системные инструкции модели преимущественно англоязычные.

## Макросы SillyTavern

Во всех переносимых полях используются только стандартные макросы:

```text
{{char}}
{{user}}
```

Реальное имя пользователя удаляется из сгенерированных полей защитной постобработкой.

## Два режима генерации

### Новая версия персонажа

1. Собирается план мира: темы, встречи, настроение, архетипы и жанровые теги.
2. `buildAdaptCharacterArial()` создаёт новую версию карты.
3. Отмеченные поля переписываются; неотмеченные принудительно восстанавливаются из исходной карты.
4. `buildUniverseArial()` создаёт `first_mes` и дополнительные приветствия по итоговой карте.

### Только новые приветствия

Этап `buildAdaptCharacterArial()` не запускается. Исходные `description`, `personality`, `scenario` и `mes_example` объединяются в канонический контекст для `buildUniverseArial()`.

Карта, аватар, теги и технические данные при этом сохраняются.

## Приоритет пользовательских параметров

Собственные тема, встреча и настроение помещаются в отдельные блоки высокого приоритета:

```text
===== UNIVERSE SETTING (HIGHEST PRIORITY) =====
===== ENCOUNTER (HIGHEST PRIORITY) =====
===== MOOD (HIGHEST PRIORITY) =====
```

Пользовательский текст рассматривается как описание, а не как системная инструкция. Встроенные команды внутри него должны игнорироваться.

Служебные значения `none`, `random` и `custom` взаимоисключающие. Обычные значения могут объединяться.

## Генерация первого сообщения

Функция: `buildUniverseArial()`.

Базовые правила:

```text
Use {{char}} for the character.
Use {{user}} for the player.
Never output the player's real display name.
Do not write actions, thoughts, biography, decisions, or dialogue for {{user}}.
End with a playable hook that invites {{user}} to respond.
```

Промпт получает:

- итоговые поля карты;
- выбранный мир, встречу и настроение;
- язык приветствия;
- размер сцены;
- последние сообщения чата как справку о речи;
- анти-слоп первого сообщения;
- активное направление первого сообщения.

Для вариантов 2–3 меняются только встреча и настроение. Карта остаётся общей.

## Адаптация карты

Функция: `buildAdaptCharacterArial()`.

Модель возвращает строгий JSON:

```json
{
  "description": "...",
  "personality": "...",
  "scenario": "...",
  "mes_example": "..."
}
```

Флаги генерации управляют четырьмя полями:

- `description`;
- `personality`;
- `scenario`;
- `mes_example`.

Снятый флаг означает **сохранить исходное значение**. Даже если модель вернула другой текст, расширение заменяет его оригиналом после разбора JSON.

### Что пересобирается в полном режиме

В зависимости от флагов и выбранного мира могут быть изменены:

- мир и эпоха;
- биография и место проживания;
- роль, занятие и связи;
- сценарий, цели и секреты;
- одежда, вид и тело;
- личность и примеры речи.

Аватар моделью не генерируется и не заменяется.

## Шаблон description

По умолчанию используется структурированная анкета со следующими разделами:

```text
World / Time Period / World Details
Scenario
Appearance
Starting Outfit
Origin
Residence
Connections
Goal
Secret
Personality

```

Пользовательский шаблон заменяет стандартную структуру `description`, но не отключает обязательный JSON-формат результата.

## Направления

Встроены направления:

- Японский визуальный роман;
- Западный TTRPG-лист;
- Корейская дорама;
- Литературный RP;
- Редактура существующей карточки.

У каждого обычного направления есть два независимых слоя:

```text
ACTIVE DIRECTION INSTRUCTIONS (CHARACTER CARD)
ACTIVE DIRECTION INSTRUCTIONS (FIRST MESSAGE)
```

Обычные направления взаимоисключающие.

### Парная редактура

**Редактура существующей карточки** может работать отдельно или вместе с одним обычным направлением. Её промпт записывается в `antiSlopCard` и виден пользователю в поле **«Анти-слоп карты персонажа»**.

Правила редактуры:

```text
НЕ переписывать карточку с нуля.
Убрать воду и клише.
Конкретизировать расплывчатые детали.
Сохранить авторский голос и ключевые черты.
Не добавлять отсутствующие факты.
```

Если автоматически вставленный текст не был изменён, отключение редактуры удаляет его. Если пользователь отредактировал поле, его текст сохраняется.

## Анти-слоп

Два независимых поля:

- `antiSlopCard` — требования к карте;
- `antiSlopFirstMes` — требования к первым сообщениям.

Они добавляются отдельными блоками и не заменяют стандартные инструкции формата и макросов.

## Свой промпт адаптации

Собственный промпт поддерживает подстановки:

```text
{{char}}
{{user}}
{{story}}
{{description}}
{{personality}}
{{scenario}}
{{mes_example}}
{{adapt_size}}
{{adapt_language}}
```

Он должен потребовать JSON с ключами `description`, `personality`, `scenario`, `mes_example`. Направление карты и анти-слоп продолжают добавляться к пользовательскому промпту.

## Механика «Яйцекладка»

При выборе `encounterTypes.oviposition` карта должна содержать совместимую физиологию `{{char}}`. Если исходный персонаж несовместим, полный режим меняет вид, тело, анатомию и сценарную логику.

Ответ проверяется функцией `validateAdaptedCard()`. Результат отклоняется, если механика не отражена или персонаж остался обычным человеком без совместимой физиологии.

## Осмыслитель карты

Функции:

```text
buildCardAnalyzerPrompt()
showCardAnalyzerModal()
```

Осмыслитель передаёт модели текущие:

- имя;
- `description`;
- `personality`;
- `scenario`;
- `mes_example`.

Доступны русская и английская версии аналитического промпта. Ответ должен содержать 3–5 рекомендаций в формате:

```text
Название сеттинга
Почему именно здесь / Why here
Ключевой конфликт / Key conflict
Пример сцены / Scene example
```

Анализ основан на незакрытых арках, внутренних противоречиях, потерях, страхах и нереализованном эмоциональном потенциале. Запрещены рекомендации только по профессии или боевой силе.

Осмыслитель не изменяет карту, не сохраняет рекомендации в коллекцию и не участвует в экспорте.

## Экспорт и сохранение канона

В режиме новых приветствий экспорт сохраняет:

- исходное имя;
- поля карты;
- теги;
- расширения;
- системный промпт и post-history instructions;
- character book;
- версию и доступные технические поля;
- исходный аватар для PNG.

В полном режиме к расширениям добавляется `extensions.seaofuniverses` с метаданными выбранного мира. Техническое имя сохранено ради обратной совместимости и не совпадает с новым отображаемым названием «Зазеркалье».

## Отмена и безопасность выполнения

Генерация использует `AbortController`. Закрытие окна Осмыслителя отменяет его запрос. Обычная генерация также поддерживает отмену через интерфейс прогресса.

Пользовательские описания санитизируются и ограничиваются по длине. Ответ адаптации разбирается как JSON; лишние code fences и текст вокруг объекта удаляются перед `JSON.parse()`.
