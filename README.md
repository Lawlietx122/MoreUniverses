# 🌌 Другая Вселенная

Русская адаптация расширения **Another Universe** для
[SillyTavern](https://github.com/SillyTavern/SillyTavern). Расширение переносит выбранного
персонажа в альтернативную вселенную, собирает обновлённую анкету и создаёт новое
приветствие для начала ролевой игры.

Основано на оригинальном проекте
[mangkoodkung/another-universe](https://github.com/mangkoodkung/another-universe).

## Что создаёт расширение

Результат состоит из двух разных частей:

- **Анкета персонажа** — `description`, `personality`, `scenario` и `mes_example`.
  Эти поля описывают новую AU-версию персонажа и попадают в экспортируемую карточку.
- **Приветствие** — `first_mes`, то есть первое сообщение новой карточки. Оно задаёт
  стартовую сцену и приглашает `{{user}}` продолжить игру.

В окне результата обе части показываются сразу: сначала превью анкеты, затем превью
приветствия. Кнопка **«Редактировать»** позволяет изменить и анкету, и приветствие перед
экспортом.

## Отличия русской версии

- Полностью русский интерфейс, уведомления, подсказки и README.
- Приветствие генерируется на русском языке.
- Двухэтапная сборка: сначала AU-анкета, затем приветствие на её основе.
- Объединённое превью **«Анкета персонажа + Приветствие»**.
- Можно выбрать несколько тем, встреч и настроений одновременно.
- Четыре размера анкеты и четыре размера приветствия.
- Настраиваемый промпт адаптации анкеты.
- Настоящий assistant prefill через API SillyTavern. Он не записывается в чат и снижает
  вероятность пустого ответа/ошибки `no candidates` у чувствительных провайдеров.
- Исправлена сборка `themeId` в Character Card V2 для экспорта PNG и JSON.
- Русские имена сохраняются в названиях экспортируемых файлов.
- Имена персонажей и ответы модели экранируются перед вставкой в интерфейс.

## Возможности

- Комбинации тем вселенной, способов встречи и эмоционального настроения.
- Собственные тема, встреча и настроение — до 1 000 символов на поле.
- Сохранение до 50 обычных записей в галерее; избранные записи не удаляются автоматически.
- Редактирование анкеты и приветствия перед экспортом.
- Экспорт карточки в двух форматах:
  - **PNG Character Card V2** — можно перетащить обратно в SillyTavern;
  - **JSON Character Card V2** — переносимый вариант без изображения.
- Экспорт приветствия как длинной или короткой картинки.
- Поддержка мобильного и настольного интерфейса.

## Требования

- Актуальная версия SillyTavern.
- Настроенное текстовое API в SillyTavern.
- Выбранный персонаж с доступной анкетой.

## Установка

### Через интерфейс SillyTavern

1. Откройте панель **Extensions**.
2. Выберите установку расширения по URL.
3. Укажите:

   ```text
   https://github.com/Shino4ka/another-shinoverse
   ```

4. Обновите страницу SillyTavern после установки.

### Вручную

Клонируйте репозиторий в каталог сторонних расширений:

```bash
cd SillyTavern/public/scripts/extensions/third-party
git clone https://github.com/Shino4ka/another-shinoverse.git
```

Итоговый путь должен выглядеть так:

```text
SillyTavern/public/scripts/extensions/third-party/another-shinoverse
```

Затем перезапустите SillyTavern или обновите страницу.

## Использование

1. Выберите персонажа в SillyTavern.
2. Нажмите кнопку `🌌` в меню чата.
3. Выберите одну или несколько тем, встреч и настроений.
4. Настройте размеры анкеты и приветствия.
5. При необходимости задайте свой промпт адаптации.
6. Нажмите **«Собрать карточку»**.
7. Проверьте результат в двух разделах:
   - **Анкета персонажа**;
   - **Приветствие**.
8. Отредактируйте результат или экспортируйте его в PNG/JSON.

## Экспорт Character Card V2

PNG и JSON собираются из одного объекта данных, поэтому содержимое обоих форматов
совпадает. В карточку записываются:

- `name` — имя AU-версии персонажа;
- `description` — адаптированная анкета;
- `personality` — адаптированная личность;
- `scenario` — адаптированный сценарий;
- `mes_example` — примеры речи;
- `first_mes` — сгенерированное приветствие;
- `tags` — исходные теги и метки выбранных AU-параметров;
- `extensions.another_universe` — `themeId`, дополнительные выбранные ID и служебные
  метаданные расширения.

Имена пользователя и персонажа в переносимых полях заменяются на макросы `{{user}}` и
`{{char}}`, где это необходимо.

## Галерея

Каждая успешная сборка автоматически сохраняется в галерее. Для записи доступны:

- просмотр анкеты и приветствия;
- добавление в избранное;
- экспорт PNG;
- экспорт JSON;
- удаление;
- резервная копия галереи в TXT.

Обычные старые записи удаляются при превышении лимита. Избранные записи защищены от
автоматического удаления.

## Если генерация не работает

- Убедитесь, что расширение включено и персонаж выбран.
- Проверьте подключение к API и лимит токенов.
- Для больших анкет и приветствий увеличьте доступный контекст модели.
- Если API возвращает `no candidates`, попробуйте менее чувствительную модель или смягчите
  формулировки темы. Prefill уменьшает вероятность такой ошибки, но фильтрация выполняется
  на стороне провайдера.
- Если модель вернула невалидный JSON анкеты, повторите сборку или упростите собственный
  промпт адаптации.
