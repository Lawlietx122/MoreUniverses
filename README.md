# 🌌 Море Вселенных

Расширение для [SillyTavern](https://github.com/SillyTavern/SillyTavern), которое переносит
выбранного персонажа в альтернативную вселенную, собирает обновлённую анкету и создаёт
одно или несколько стартовых приветствий для начала ролевой игры.

Идея берет начало из оригинального проекта
[Another Universe](https://github.com/Shino4ka/another-shinoverse). Код, логика и интерфейс
были переосмыслены и расширены.

**Автор:** [V Jewel — t.me/TavernSilly](https://t.me/TavernSilly)

---

## Что создаёт расширение

Результат состоит из двух частей:

- **AU-анкета** — `description`, `personality`, `scenario`, `mes_example`.
  Новая версия персонажа, адаптированная под выбранный сеттинг, тропы и атмосферу.
- **Приветствие** — `first_mes` и до двух дополнительных вариантов
  (`alternate_greetings`). Каждое приветствие — стартовая сцена с игровым крючком.

В окне результата видны обе части. Кнопка **«Редактировать»** позволяет изменить анкету,
приветствие и альтернативные приветствия перед экспортом.

## Интерфейс

Окно настроек разделено на три вкладки для компактности.

### Вкладка 1 — «Сюжет и Мир»

| Поле | Описание |
|------|----------|
| **Сеттинг (мир / эпоха)** | Где происходит действие. Средневековье, киберпанк, космос, академия и т. д. Группируется по категориям «Сеттинг» и «Жанровая рамка». |
| **Жанровые теги** | Жанровая рамка истории. Любое количество в интерфейсе, но в итоговую карточку попадёт максимум 3. |
| **Троп и завязка** | Как персонажи встречаются. Нарративные тропы, завязки сцен, типы отношений, фантастические механизмы, 18+ тропы (включая яйцекладку). |
| **Тон и атмосфера** | Как ощущается сцена. Тон повествования, атмосфера, эмоциональный регистр. |
| **Архетип и динамика** | Типаж героя и характер связи. Яндере, цундере, куудере, одержимость, ревность, враги→любовники и др. |
| **Описание своей вселенной** | Кастомный контекст мира. Добавляется в промпт как блок `WARNING NOTE` перед данными персонажа. |
| **Варианты приветствия (1–3)** | До трёх приветствий с разными тропами/атмосферой. Вариант 1 → `first_mes`, варианты 2–3 → `alternate_greetings`. |
| **Скрыть стандартные варианты** | Чекбокс, скрывающий встроенные опции во всех выпадающих списках. |

**Комбо-компоненты вариантов 2/3:** поле поиска с фильтрацией в реальном времени. Если
введённый текст не совпадает ни с одной опцией — появляется пункт «✏️ Использовать: "..."».
Кастомные значения сохраняются в `localStorage` (`sou_custom_encounter_v2`,
`sou_custom_mood_v2`, `sou_custom_encounter_v3`, `sou_custom_mood_v3`) и отображаются с
иконкой 🌟. Удаление — кнопкой ×.

### Вкладка 2 — «Генерация»

| Блок | Описание |
|------|----------|
| **Размер приветствия** | Краткая / Стандартная / Подробная / Максимальная. |
| **Язык первого сообщения** | Русский / Английский / Как в анкете. |
| **Язык анкеты / Размер анкеты** | Язык и объём адаптированной анкеты. |
| **⚙️ Что генерировать** | Чекбоксы: Personality, Scenario, Message Example. Снятие чекбокса → поле не генерируется и остаётся пустым. Для каждого поля — кнопка «✏️ Свой промт» (сохранение в `localStorage`, маркер «🌟 кастомный промт активен»). |
| **📋 Структура карточки персонажа** | Раскрывающийся список всех полей, из которых собирается карточка. |
| **📝 Шаблон и промпты анкеты** | Редактируемый шаблон описания (заменяет стандартный), свой промпт адаптации, сохранение/загрузка шаблонов. |
| **🚫 Анти-слоп** | Два поля: «Анти-слоп карты персонажа» и «Анти-слоп первого сообщения». Инструкции модели: что писать и чего избегать. |

### Вкладка 3 — «Настройки»

| Настройка | Описание |
|-----------|----------|
| **Тема оформления** | Тёмная / Светлая / Серая. Применяется через CSS-переменные. |
| **Акцентный цвет** | Color-picker. По умолчанию `#DEB887` (BurlyWood). Применяется к чекбоксам, бордерам, кнопкам, выделению. |
| **Creator** | Никнейм или псевдоним. Если пусто — поле `creator` в карточке остаётся пустым. |

## Процесс генерации

При нажатии **«Собрать карточку»**:

1. **Адаптация AU-анкеты** — модель создаёт новую анкету персонажа под выбранный
   сеттинг, сохраняя характер, голос и поведенческую логику оригинала.
2. **Генерация приветствий** — для каждого варианта (1–3) модель пишет стартовую сцену
   на основе уже адаптированной анкеты. AU-анкета генерируется один раз, меняются только
   троп/атмосфера.

Прогресс отображается поэтапно: список этапов с галочками (✅/⏳/⬜) и прогресс-бар.
Анимация — пульсирующий красный фейерверк (`firework-pulse`), текст «Уходим в
зазеркалье...».

## Экспорт Character Card V2

PNG и JSON собираются из одного объекта данных. В карточку записываются:

| Поле | Содержимое |
|------|------------|
| `name` | Имя AU-версии персонажа |
| `description` | Адаптированная анкета |
| `personality` | Адаптированная личность (если включено) |
| `scenario` | Адаптированный сценарий (если включено) |
| `mes_example` | Примеры речи (если включено) |
| `first_mes` | Сгенерированное приветствие (вариант 1) |
| `alternate_greetings` | Дополнительные варианты приветствий (2, 3) |
| `tags` | Максимум 3 тега (жёсткое ограничение) |
| `creator` | Значение из поля Creator (или пустая строка) |
| `notes` | Всегда пустая строка |
| `extensions.seaofuniverses` | Метаданные: themeId, encounterIds, moodIds, archetypeIds, genreTagIds и др. |

Имена пользователя и персонажа в переносимых полях заменяются на макросы `{{user}}` и
`{{char}}`.

## Галерея

Каждая успешная сборка сохраняется в галерее (до 50 записей, избранные не удаляются).

- Просмотр анкеты, приветствия и альтернативных приветствий;
- Добавление в избранное ⭐;
- Экспорт PNG 🎴;
- Экспорт JSON 🗂️;
- Удаление 🗑️;
- Резервная копия галереи в TXT.

## Сохранение данных

| Тип данных | Где хранится |
|------------|--------------|
| Настройки расширения | `extension_settings` (SillyTavern) |
| Кастомные троп/атмосфера вариантов 2/3 | `localStorage` (`sou_custom_*`) |
| Кастомные промпты полей | `localStorage` (`sou_prompt_*`) |
| Тема оформления и акцентный цвет | `extension_settings` |

## Установка

### Через интерфейс SillyTavern

1. Откройте панель **Extensions**.
2. Выберите установку расширения по URL.
3. Укажите:

   ```text
   https://github.com/Shino4ka/sea-of-universes
   ```

4. Обновите страницу SillyTavern после установки.

### Вручную

Клонируйте репозиторий в каталог сторонних расширений:

```bash
cd SillyTavern/public/scripts/extensions/third-party
git clone https://github.com/Shino4ka/sea-of-universes.git
```

Итоговый путь:

```text
SillyTavern/public/scripts/extensions/third-party/sea-of-universes
```

Затем перезапустите SillyTavern или обновите страницу.

## Краткое руководство

1. Включите расширение в панели Extensions.
2. Выберите персонажа в SillyTavern.
3. Нажмите кнопку `🌌` в меню чата.
4. На вкладке **«Сюжет и Мир»** выберите сеттинг, жанры, тропы, атмосферу и архетип.
5. При необходимости опишите свою вселенную и настройте варианты приветствий.
6. На вкладке **«Генерация»** выберите размер, язык и что генерировать.
7. На вкладке **«Настройки»** укажите автора карточки (необязательно).
8. Нажмите **«Собрать карточку»**.
9. Проверьте результат, отредактируйте при необходимости.
10. Экспортируйте в PNG или JSON.

## Требования

- Актуальная версия SillyTavern.
- Настроенное текстовое API.
- Выбранный персонаж с заполненной анкетой.

## Если генерация не работает

- Убедитесь, что расширение включено и персонаж выбран.
- Проверьте подключение к API и лимит токенов.
- Для больших анкет и приветствий увеличьте доступный контекст модели.
- Если API возвращает `no candidates` — попробуйте менее чувствительную модель или
  смягчите формулировки. Prefill снижает вероятность этой ошибки.
- Если модель вернула невалидный JSON — повторите сборку или упростите кастомный промпт.
- Проверьте, что поля «Свой троп/атмосфера» не пустые, если выбран соответствующий пункт.
